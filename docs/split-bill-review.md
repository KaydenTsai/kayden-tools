# Split Bill (SnapSplit) 產品 Review 與商業化轉型建議

**日期:** 2025-12-26
**狀態:** 現況分析與未來規劃

## 1. 現況分析 (Current State)

目前的 Split Bill 是一個 **「本地優先 (Local-First)、無後端、快照式分享」** 的工具。

*   **優勢:** 隱私性高、無須註冊、離線可用、操作流暢。
*   **劣勢:** 
    *   **協作斷層:** 分享連結僅為當下快照 (Snapshot)，無法雙向同步修改。
    *   **資料風險:** 資料僅存於瀏覽器 LocalStorage，容易遺失。
    *   **輸入負擔:** 缺乏自動化工具 (如 OCR)，依賴全手動輸入。

---

## 2. 商業化關鍵：使用者帳戶與登入機制 (User Accounts & Auth)

為了轉型為收費產品 (SaaS)，**「雲端同步與協作」** 是最核心的付費價值。這需要引入完整的身份驗證系統。

### 2.1 混合儲存策略 (Hybrid Storage Strategy)
為了保留現有的「無痛試用」體驗，建議採用混合模式：

1.  **訪客模式 (Guest / Local):** 
    *   維持現狀。
    *   資料存於 LocalStorage。
    *   功能限制：無法即時協作，僅能生成快照連結。
2.  **登入模式 (Auth / Cloud):**
    *   **Pro 功能基礎。**
    *   資料同步至雲端資料庫 (Supabase/Firebase)。
    *   支援多裝置存取、即時協作。

### 2.2 登入機制設計 (Authentication Flow)

*   **登入方式:** 
    *   首選 **Social Login (Google / Apple)**。減少輸入帳密門檻，適合行動裝置。
    *   Email Magic Link (備選)。
*   **進入點 (Entry Points):**
    *   **Top Bar:** 右上角新增「登入/註冊」按鈕 (取代目前的靜態顯示)。
    *   **Feature Gating (功能觸發):** 當使用者點擊「開啟即時協作」或「OCR 掃描」時，彈出登入引導視窗 (Auth Modal)。
*   **資料遷移 (Data Migration):**
    *   當使用者從「訪客」轉為「登入」狀態時，系統應詢問：「是否將本地的 3 筆帳單同步至雲端？」確保資料不丟失。

## 3. UI/UX 改進建議 (UI/UX Improvements)

針對商業化轉型與現有體驗的不足，提出以下 UI 具體改進方案：

### 3.1 導航與佈局 (Navigation & Layout)

*   **頂部導航列 (App Bar) 改造:**
    *   **現狀:** 僅顯示標題。
    *   **改進:** 
        *   **左側:** 返回 / Home。
        *   **中間:** 帳單名稱 (點擊可編輯)。
        *   **右側:** **使用者頭像 (User Avatar)**。未登入顯示「登入」文字或人像 placeholder。
        *   **同步狀態燈:** 在標題旁顯示小圖示 (雲朵=已同步, 驚嘆號=未同步/本地)。
*   **底部導航 (Mobile Optimization):**
    *   **建議:** 考慮將目前的 Top Tabs (`記錄`、`明細`、`結算`) 改為 **底部導航列 (Bottom Navigation Bar)**。
    *   **原因:** 手機單手操作更友善，符合主流 App (如 Splitwise) 的操作習慣。

### 3.2 協作與即時性 UI (Collaboration UI)

*   **成員清單視覺化:**
    *   在帳單標題下方顯示一排成員的小圓頭像 (Avatars)。
    *   **Active Indicator:** 當某位成員正在查看或編輯時，頭像亮起或顯示綠點 (Live Presence)。
*   **活動紀錄 (Activity Log):**
    *   新增一個「動態」分頁或收合面板，顯示：「Kayden 新增了午餐 $300」、「Alice 修改了計程車費」。
    *   **價值:** 增加信任感，讓付費更有感。

### 3.3 商業化功能 UI (Premium Features UI)

*   **OCR 掃描入口 (高價值):**
    *   在「新增消費」的介面中，新增顯眼的 **「相機/掃描」Icon**。
    *   **Upsell UI:** 如果是免費用戶，點擊後彈出精美的 Paywall Modal，展示「AI 自動辨識，省去手動輸入時間」的預覽動畫。
*   **VIP 標示:**
    *   Pro 用戶在頭像旁顯示「皇冠」或「Pro」徽章。
    *   匯出的圖片報表移除浮水印，並允許自訂主題色。

### 3.4 現有體驗優化 (General Polish)

*   **空狀態 (Empty States) 優化:**
    *   目前的 `BillListView` 空狀態較為單調。
    *   **改進:** 使用插圖 (Illustration) 配合引導文字。例如：「建立你的第一筆旅費，或是 **掃描收據 (Pro)** 來快速開始」。
*   **輸入優化 (Input Experience):**
    *   **金額鍵盤:** 在手機版輸入金額時，確保彈出的是純數字鍵盤 (Number Pad)，而非全鍵盤。
    *   **快速選人:** 在 `ItemizedExpense` 模式，選擇參與者時，建議使用「點擊頭像」的網格佈局 (Grid)，而非下拉選單或純文字列表，點選效率更高。

## 4. 總結 (Summary)

要從工具變產品，**UI 必須從「個人計算機」演化為「多人聊天室」的感覺**。

1.  **Login 是手段，Sync 是目的:** 登入介面要輕量，重點是告訴用戶「登入後你的資料就不會見了」。
2.  **付費點顯性化:** 將 OCR、匯出報表、多幣別等按鈕直接做在 UI 上，點擊後再跳出付費牆 (Paywall)，而非隱藏這些功能。
